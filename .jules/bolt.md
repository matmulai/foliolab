## 2025-02-20 - Combine RegExp Patterns for Efficient String Replacement
**Learning:** Sequential calls to `String.prototype.replace()` within an array iteration using different RegExp patterns are inefficient. It requires the JavaScript engine to perform M sequential passes (where M is the number of patterns) over a string of length N, resulting in O(M * N) complexity.
**Action:** Pre-compile related regex patterns into a single combined regex using `new RegExp(patterns.map(p => p.source).join('|'), 'gi')`. This reduces redundant intermediate string creation and allows a single O(N) replacement pass, significantly improving string processing performance. Ensure existing patterns do not rely on exact backreference numbering (e.g., `\1`), as combining patterns alters capture group indices and breaks backreferences.
