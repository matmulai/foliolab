## 2024-05-18 - SSRF Filter Bypass with IPv4-mapped IPv6 Addresses
**Vulnerability:** The SSRF mitigation function `isPrivateIP` could be bypassed by using an IPv4-mapped IPv6 address like `::ffff:127.0.0.1` or `::ffff:169.254.169.254`. Node.js's native HTTP clients and libraries (like `rss-parser`) correctly resolve and connect to these addresses as the underlying IPv4 address, but the regex-based blocklist (which checks for `127.`, `169.254.`, etc.) would fail to match because of the `::ffff:` prefix.
**Learning:** `net.isIP('::ffff:127.0.0.1')` returns true (it evaluates as a valid IPv6 address). However, the subsequent validation relies on regular expressions designed solely for IPv4 and specific IPv6 loopbacks (`::1`). The prefix causes a false negative in the blocklist logic, allowing direct internal network access.
**Prevention:** Always normalize IP addresses (by explicitly stripping prefixes like `::ffff:` or resolving them down to their core representation) before testing them against regex blocklists. A more robust alternative to regex is using libraries like `ipaddr.js` to parse and check IP ranges securely.
